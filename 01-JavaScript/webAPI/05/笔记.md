> 给元素注册事件的方式
```js
     
     1. 给元素注册事件的方式

        通过 '. + on事件名称' ;

        注意：
            ✔ 通过 on的方式给元素注册事件，如果注册的是相同的事件，那么最后的事件会将前面的事件覆盖掉


     2. 通过事件监听的方式给元素注册事件

         步骤：

             ✔ 先获取事件源

             ✔ 通过 addEventListener() 方法给元素注册事件

             ✔ 在  addEventListener(参数1, 参数2, 参数3)

                  参数1： 代表具体的事件类型     注意： 不能使用 'on'

                  参数2:  代表的就是一个处理程序

        注意：
             通过 addEventListener给元素注册的事件，不会覆盖。


      3. 兼容IE低版本浏览器（8）

          通过  attachEvent(参数1, 参数2) 方式

          参数1: 代表事件类型   注意： 在该方法中，事件类型需要添加 'on'

          参数2: 代表处理程序


      4. 如何保证能在IE低版本浏览器中执行IE兼容的代码

          <!--[if IE 8]>

              <script> js 代码 </script>

          <![endif]-->

```
> 如何给元素移除事件
```js
     
    1. 如果通过 'on' 的方式给元素注册的事件，那么在移除对应的事件的时候怎么做？

        dom.事件类型 = null;


    2. 如果通过 'addEventListener'的方式给元素注册的事件,那么怎么移除？

        通过 'removeEventListener(参数1，参数2)' 的方式将对应的事件移除掉

        参数1: 要被移除的事件类型

        参数2: 要被移除的处理程序名称


        注意事项：
             1. 在注册事件的时候，处理程序必须是一个命名函数。

        例如：
            
              btn.addEventListener('click', fn);
              function fn() {
                  alert(1);
              }
              //移除对应的该事件
              btn.removeEventListener('click',fn);
    
    3. 如果通过 attachEvent() 方式注册的事件，那么需要通过 detachEvent() 将对应的事件移除掉

         注意：
             1. detachEvent(参数1, 参数2)

             2. detachEvent() 在移除事件的时候，也需要处理程序是一个命名函数

             3. 使用方式与 removeEventListener 一样。

```

> 事件的三个阶段
```js
    
    1. addEventListener 中第三个参数是一个布尔类型的结果

        1.1 :  如果 addEventListener 第三个参数是false（默认值），那么就会出现事件冒泡的现象

        1.2  ：如果 addEventListener 第三个参数是true，那么会出现事件捕获的现象


    2. 事件冒泡： 事件会从当前事件源处开始向外传递


    3. 事件捕获： 事件会从外向内传递，传递到点击当前的事件源为止


    4. 事件执行： 事件到事件源处执行完了（点击的当前真正事件源）


    注意：
         ✔ 通过 onclick的方式只能出现 事件冒泡，没有调用执行捕获阶段 （不支持第三个参数的设置）

         ✔ 通过 attachEvent的方式注册事件 出现事件冒泡，没有调用执行捕获阶段（不支持第三个参数的设置）   
```

