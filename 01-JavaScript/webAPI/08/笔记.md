> 利用移动端事件封装一个点击事件和滑动事件（轻滑事件）
```js

     ☞ 点击事件思路： 手指按下去时候的位置 和 手指离开时候的位置一样

     ☞ 如何获取手指按下时候的位置信息，通过事件对象参数获取

     ☞ 移动端事件对象参数介绍：

         事件对象参数.touches :   获取在屏幕上的手指信息

         事件对象参数.targetTouches :  获取位于页面中点击元素身上的手指信息      

         事件对象参数.changedTouches : 获取当手指离开屏幕时候的手指信息

         总结：
             1. 将屏幕上的所有手指信息以伪数组的形式保存下来

             2. 该数组中的每一个值都是以对象的形式保存数据的。【每根手指都是一个对象】

             3. length属性是用来获取手指的个数。

             4. clientX: 获取手指距离视口的水平坐标

             5. clientY：获取手指距离视口的垂直坐标

             6. pageX : 获取手指距离视口的水平坐标 （如果页面中出现滚动条，就包括了滚动条滚动出去的位置）
```

> 封装一个手指滑动的事件
```js

    滑动事件思路： 
    
        1.按下时候的位置 和 离开时候的位置是否一样

        2.如果是水平滑动：
              
              水平的距离 > 垂直距离

             
              从左向右滑动：     开始位置 < 结束位置

              从右向左滑动：     开始位置 > 结束位置


        3. 如果是垂直滑动

               垂直距离 > 水平距离

               从上向下滑动：   开始位置 < 结束位置

               从下向上滑动：   开始位置 > 结束位置
```

# 拓展H5中API
> 文件读取
```js
    上传功能： 文件读取

    步骤：  

        1. 准备一个上传控件  (<input type="file">)

        2. 给当前控件注册一个change事件

        3. 在change事件中获取上传的资源信息

        4. 用户上传的资源都在 files【伪数组，每一个资源都是以对象的形式保存】

        5. 要读取文件的时候，需要创建一个读取器

            例如： var reader = new FileReader();


            事件： onload （当资源完全读取完成后就会触发的一个事件）

            属性： result （获取最后的读取结果）


            方法： readAsText()  :    以文本的形式读取文件（获取文件中的文字信息）

                  readAsDataURL() :   将文件的路径读取处理

        6. 注意： 不能直接获取读取结果，当前资源被读取完成后才能获取读取结果  
```
> 本地存储
```js

    ☞ sessionStorage
         
         保存数据
         sessionStorage.setItem(键, 值);

         获取数据
         sessionStorage.getItem(键);

         删除数据
         sessionStorage.removeItem(键);

         清空所有的数据
         sessionStorage.clear();


         总结：
             1. 保存的数据是临时数据（页面关闭后，所有的数据都被自动清空）
             2. sessionStorage保存的值只能在当前页面中访问
             3. 保存数据的大小大约在 5M

    ☞ localStorage
        localStorage.setItem(键, 值);
        localStorage.getItem(键);
        localStorage.removeItem(键);
        localStorage.clear();
         总结：
             1. 保存的数据是持久化的数据（页面关闭后，数据依然存在）；
             2. localStorage 保存的数据可以在其他页面中访问
             3. localStorage保存数据的大小大约在 20M

```
